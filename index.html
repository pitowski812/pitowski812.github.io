Pep①Ai — RAG assistant with fail-closed safety architecture

Designer / builder · hybrid retrieval (BM25 + vectors + rank fusion) · local embeddings · multi-stage safety gates

A domain research assistant grounded in a curated evidence corpus (PubMed ingestion, community sources, audio-transcript pipeline via Whisper) — built for a domain where a wrong answer is a safety problem, not a typo.

  • Three-state safety contract: every checker distinguishes passed / blocked / checker-unavailable — an internal guard failure can never masquerade as “no violation found.” Deterministic guards down → the answer is withheld with a distinct degraded message; outages log greppable markers for alerting.
  • Defense in depth: pre-generation intent screening, deterministic bounds checking on generated numbers, claim-level faithfulness gating with a different-vendor LLM judge (and a circuit breaker so a judge outage reads as an outage, not a verdict).
  • Evidence transparency: formal literature and community-reported practice are separate, labeled lanes with exact attribution — community signal is preserved but never silently promoted to clinical fact.
  • Evaluated, not vibes: frozen retrieval benchmark + a 50-case answer-quality golden set judged by a different model vendor; route-level exception-injection tests prove the failure behavior.
